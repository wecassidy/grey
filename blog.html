Why doesn't this where binding work? (Haskell)

by Tungsten, May 22, 2014, 6:07 pm

Code:
  1. thing :: Int -> Int -> String
  2. thing a b
  3.     | thwip a b == 0 = lose
  4.     | thwip a b == 1 = win
  5.     | otherwise      = tie
  6.     where thwip = a `mod` b
  7.           lose = "You win."
  8.           win = "I win!"
  9.           tie = "It's a tie. Let's play again!"

Despite this code's embarrassing uselessness, I'm pretty sure it should work. Unfortunately, it doesn't. When I try to run it in GHCi/SublimeREPL Haskell, this is what happens:
Code:
>>> :l thwop
[1 of 1] Compiling Main             ( thwop.hs, interpreted )

thwop.hs:7:24: parse error on input `='
Failed, modules loaded: none.


Does anybody have any idea why this is happening? The where binding

  1.    bmiTell :: (RealFloat a) => a -> a -> String

  3.     | bmi <= skinny = "You're underweight, you emo, you!"

  5.     | bmi <= fat    = "You're fat! Lose some weight, fatty!"  
  6.     | otherwise     = "You're a whale, congratulations!"  
  7.     where bmi = weight / height ^ 2  
  8.           skinny = 18.5  
  9.           normal = 25.0  
  10.           fat = 30.0
